Logging is pervasive. Every function carries around a "logger" instance, which is instrumented to add extra info, as a json obj.

7:40

Our logging idea is very similar to github.com/trentm/node-bu…. But we don't let the library objects leak into the code, but use a wrapper.

7:41

We stream every log line (a json obj) to @logmatic, which is *awesome* to analyze and produce metrics.

7:43

Every server-side request has a unique id that's added to the log lines, so it possible to rebuild the entire history of a given request.

7:44

Every sanity check that fails produces a special "security warning" log event. A real time alert notifies hack attempts.
